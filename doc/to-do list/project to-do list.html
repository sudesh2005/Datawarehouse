This comprehensive list outlines the steps for building the modern SQL Data Warehouse. **Check the box to mark a task as complete.**

I. Project Planning & Design (Initial Epics)

  • Epic 1: Requirements Analysis
    • Analyze and understand the project requirements (Core: Build DW to consolidate sales data, focus on latest data, and generate documentation).
    **Note:** This epic is closed early in the project after analysis is complete.
  • Epic 2: Designing Data Architecture
    • Choose Data Management Approach (Decision: Build a Data Warehouse using the **Medallion Architecture**).
    • Design the layers of the Data Warehouse (Define purpose, load method, rules for Bronze, Silver, Gold).
    • Draw a Data Architecture Diagram ( [Image of Medallion Data Architecture] ).
  • Epic 3: Project Initialization
    • Create detailed project tasks.
    • Define the Naming Conventions (**Snake Case** for all objects, prefixes like `dw_` and `load_`).
    • Create Git Repository and structure (Folders for data_sets, documents, scripts, and tests).
    • Create the `data_warehouse` database and the three schemas: `bronze`, `silver`, and `gold`.

II. Building the Core Layers (ETL/ELT Development)

Epic 4: Building the Bronze Layer (Data Ingestion)

  • Analyze the source systems (CRM and ERP CSV files).
  • Write **DDL scripts** for Bronze tables (6 tables: `source_system_entity`. Include existence checks).
  • Write script for **Data Ingestion** (Full load method: `TRUNCATE TABLE` followed by `BULK INSERT`).
  • Perform **Data Validation** (Check data completeness and data type accuracy).
  • Create **Stored Procedure** (`load_bronze`).
  • Add **Instrumentation** (Messaging, error handling, duration tracking).
  • Draw Data Flow Diagram (Source files to Bronze tables).
  • Document and Commit work (DDL and Stored Procedure scripts) to Git.

Epic 5: Building the Silver Layer (Clean & Standardized Data)

  • Analyze and explore data in the Bronze layer (Identify quality issues: duplicates, spaces, invalid dates).
  • Write **DDL scripts** for Silver tables (Structure is identical to Bronze, but includes `dw_create_date`).
  • Write **Data Transformation scripts** (Cleansing, normalization, deriving columns, removing duplicates).
  • Insert cleaned data into the Silver layer (Full load: `TRUNCATE` and `INSERT`).
  • Validate the Silver data quality (Re-run quality checks).
  • Create **Stored Procedure** (`load_silver`) with full instrumentation.
  • Extend the Data Flow Diagram (Bronze tables to Silver tables).
  • Document and Commit work (DDL, Stored Procedure, and quality check scripts) to Git.

Epic 6: Building the Gold Layer (Business Ready Layer)

  • Analyze Business Objects (Identify **Customer**, **Product**, and **Sales**).
  • Code **Dimension Views** (`dim_customer`, `dim_product`).
    • Integrate data from Silver layer.
    • Apply complex business logic.
    • Generate **Surrogate Keys** (e.g., `customer_key`).
    • Rename columns to friendly, business-aligned names.
  • Code **Fact View** (`fact_sales`).
    • Perform **Data Lookup** to replace source keys with Surrogate Keys.
    • Structure columns according to fact table conventions.
  • Create final objects as **Views** (Virtual, no load process required).
  • Validate **Data Integrity** (Test joins between Fact and Dimension views).
  • Draw **Data Model Diagram** (Star Schema).
  • Create a **Data Catalog** (Documentation for business users).
  • Finalize and Commit work (Gold DDL, quality checks, and documentation) to Git.

III. Project Finalization (Completion)

  • Finalize Git Repository
    • Upload remaining documentation (Data Architecture, Project Overviews) to the `docs` folder.
    • Update the **Readme** file.
  • Completion
    • The data warehouse build is finished, enabling reporting and analytics.
